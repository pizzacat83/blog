A Useful Custom Function to Debug Firestore Security Rules

Firestore security rules provide a function debug, which logs the given value to firestore-debug.log (only when using the Firestore Emulator; it's no-op in production). But it just prints the value, with no information on its context. What if there's a function to log custom messages that explain why the request is denied?
